# バックテスト運用ルール

## walk-forward 分析

**パラメータ変更時は必ずウォークフォワード検証を行ってください。**

### 実行コマンド

```bash
npm run walk-forward
```

実行時間の目安: 15〜20分

### 実行タイミング

**実行すべきタイミング:**
- パラメータを変更する前後（変更の効果・汎化性能の検証）
- 戦略の大幅な見直し時
- 定期的な汎化性能の確認（月1〜四半期に1回程度）
- `daily-backtest` の PF が継続的に低下してきた時（過学習か市場環境変化かを切り分け）

**実行不要なタイミング:**
- 毎日（パラメータが変わらない限り結果は変わらない）
- コードのバグ修正のみの場合

### daily-backtest との違い

| | `daily-backtest` | `walk-forward` |
|---|---|---|
| 目的 | 戦略が今も有効か日々確認 | パラメータが過学習していないか検証 |
| 実行頻度 | 毎日16:30自動実行 | 手動・必要な時のみ |
| 出力 | DB保存 → ダッシュボード表示 | コンソール出力のみ |

### 過学習判定基準

| 判定 | 条件 |
|------|------|
| **堅牢** | OOS集計PF >= 1.3 かつ IS/OOS PF比 <= 2.0 |
| **要注意** | OOS集計PF >= 1.0 かつ IS/OOS PF比 <= 3.0 |
| **過学習** | OOS集計PF < 1.0 または IS/OOS PF比 > 3.0 |

「堅牢」以外のパラメータは本番適用を慎重に判断すること。
