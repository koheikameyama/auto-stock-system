# Stock Buddy 仕様書

## サービス概要

**Stock Buddy** は株式投資初心者向けのAIデータ分析ツールです。

- **キャッチコピー**: 「データを読み解く力を、あなたに」
- **ターゲット**: 投資初心者〜トレーダー志望（学びながら成長したい人）
- **対象市場**: 日本株（東証）

AIが複雑な市場データをわかりやすく整理・翻訳し、ユーザー自身が判断できる環境を提供します。専門用語も解説付きで学びながら、投資の知識が自然と身につきます。

> **重要**: 本サービスは投資助言を行いません。売買の判断はユーザー自身の責任で行ってください。AIはデータの分析・翻訳・教育に徹し、特定の銘柄の売買を推奨しません。

## コアバリュー

| 価値 | 説明 |
|------|------|
| **寄り添う** | AIが専門用語をわかりやすく翻訳。データの意味を丁寧に解説 |
| **学べる** | テクニカル分析・財務指標を解説付きで提供。使いながら投資知識が自然と身につく |
| **シンプル** | 複雑なデータをスコアとランクで直感的に把握できる |
| **自立を支援** | 判断材料を整理して提示。最終判断はユーザー自身が行う |

## 設計思想

### AIの役割: データ分析・翻訳者

「AIは答えを出さない。判断材料を整理してユーザーに差し出す」

- **AIの役割**: データの分析・翻訳・教育（テクニカル分析の解説、財務データの要約、ニュースの整理）
- **ルールの役割**: リスク要因の検出・アラート表示（赤字、高ボラティリティ、急落などの事実を警告）

### やらないこと

| 禁止事項 | 理由 |
|----------|------|
| 売買の推奨（buy/sell/hold等） | 投資助言に該当 |
| 具体的な価格予想 | 投資助言に該当 |
| 売買タイミングの指示 | 投資助言に該当 |
| AIによる判断の自動上書き（強制補正） | ユーザーの判断を代行 |
| 特定銘柄の購入提案 | 投資助言に該当 |

### 注意フラグ（旧: 強制補正）

売買推奨の上書きは行わず、事実をアラートとして表示する。

| 条件 | 表示 |
|------|------|
| 急騰銘柄（週間変化率 ≥ 30%） | 注意フラグ: 「急騰中（週間+XX%）」 |
| 赤字 + 高ボラティリティ（volatility > 50%） | 注意フラグ: 「赤字 × 高ボラティリティ」 |

## 技術スタック

| カテゴリ | 技術 |
|----------|------|
| フロントエンド | Next.js 14 (App Router), TypeScript, TailwindCSS |
| バックエンド | Next.js API Routes |
| データベース | PostgreSQL (Prisma ORM) |
| AI | OpenAI GPT-4o-mini（データ分析・翻訳）, GPT-4（チャット） |
| 認証 | NextAuth.js |
| デプロイ | Railway（自動デプロイ） |
| バッチ処理 | GitHub Actions + Python スクリプト |
| 通知 | Web Push API |
| 国際化 | next-intl（日本語） |
| ニュース | Tavily API（ニュース取得） |
| 株価データ | yfinance（Python） |

## 制限値

| 項目 | 上限 | 定義場所 |
|------|------|----------|
| ポートフォリオ銘柄数 | 100 | `lib/constants.ts` |
| ウォッチリスト銘柄数 | 100 | `lib/constants.ts` |
| 追跡銘柄数 | 10 | `lib/constants.ts` |
| データ保持期間 | 30日 | バッチ処理で定期削除 |
| DB容量上限 | 500MB | Railway制約 |

## データモデル概要

### ユーザー関連

| テーブル | 説明 |
|----------|------|
| `User` | ユーザー情報（認証、ロール、課金プラン） |
| `UserSettings` | 投資スタイル、予算、売却目標/撤退ライン |
| `Account` / `Session` | NextAuth.js 認証情報 |
| `PushSubscription` | プッシュ通知購読情報 |

### 銘柄管理

| テーブル | 説明 |
|----------|------|
| `Stock` | 銘柄マスタ（株価、財務指標、業績データ） |
| `PortfolioStock` | ポートフォリオ（保有銘柄 + リスクフラグ） |
| `WatchlistStock` | ウォッチリスト（気になる銘柄 + 価格アラート） |
| `TrackedStock` | 追跡銘柄（株価のみ追跡、AI分析なし） |
| `Transaction` | 取引履歴（買い/売り） |

### AI分析

| テーブル | 説明 |
|----------|------|
| `StockAnalysis` | 銘柄分析（健全性スコア、リスクフラグ） |
| `StockReport` | 銘柄分析レポート（テクニカルスコア、財務健全性ランク、注意フラグ） |
| `DailyHighlight` | ユーザー別 今日の注目データ（客観条件でピックアップ） |
| `PortfolioOverallAnalysis` | 市場環境サマリー |

### 市場データ

| テーブル | 説明 |
|----------|------|
| `MarketNews` | マーケットニュース（JP/US） |
| `SectorTrend` | セクタートレンド（ニュース + 株価統合スコア） |
| `DailyFeaturedStock` | 今日の注目データ（全ユーザー共通） |
| `DailyMarketMover` | 日次上昇/下落ランキング |
| `PortfolioSnapshot` | 資産スナップショット（日次記録） |

### スクリーニング

| テーブル | 説明 |
|----------|------|
| `ScreeningPreset` | プリセットスクリーニング条件マスタ（高配当・割安・成長株等） |

### その他

| テーブル | 説明 |
|----------|------|
| `Notification` | アプリ内通知 |
| `StockRequest` | 銘柄追加リクエスト |

## 日付の取り扱い

- **DB保存**: UTC形式
- **日付境界**: JST 00:00:00（日本時間の深夜0時）
- **日付操作**: dayjs + timezone プラグイン
- **共通ユーティリティ**: `lib/date-utils.ts`（`getTodayForDB()`, `getDaysAgoForDB()`）

## 認証方式

- NextAuth.js によるセッションベース認証
- ロール: `user`（一般）/ `admin`（管理者）
- CRON認証: `CRON_SECRET` ヘッダーによるバッチ処理認証
- 利用規約/プライバシーポリシーの同意管理
