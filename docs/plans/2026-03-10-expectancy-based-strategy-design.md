# 期待値ベース戦略への移行設計

## 概要

「勝率70%・コツコツ利確」から「正の期待値・損小利大」ベースへの戦略コンセプト移行。
スイングトレード × トレンドフォローを主軸とする。

## 背景

現行の「勝率70%」コンセプトは、小さく利確して大きく負ける「コツコツドカン」のリスクを構造的に抱えている。勝率70%でも利確2%・損切り5%なら期待値はマイナス。プロの自動売買で重要なのは勝率ではなく期待値（Expectancy）。

---

## 1. プロダクトコンセプト

### Before
> 「コツコツ確実に、毎日勝つ」
> 70%以上の勝率を目指す。数%の利確を積み重ねる。

### After
> 「損小利大で、期待値を積み上げる」
> 正の期待値を持つトレードだけを執行する。勝率ではなく、1トレードあたりの期待値とリスクリワード比を最適化し、トレンドに乗って利益を伸ばす。

### コアバリュー変更

| 項目 | Before | After |
|------|--------|-------|
| 目標指標 | 勝率70%以上 | 期待値プラス（PF 1.5以上） |
| 利益戦略 | 数%の小さな利確を積み重ね | トレンドに乗り利益を伸ばす（損小利大） |
| 勝率の位置づけ | 第一KPI | 参考指標（40-55%を想定） |
| 利確方針 | 固定+3%で確定 | トレーリングストップで上値を追う |
| 損切り方針 | ATR×1.0（最大3%） | ATR×1.0（変更なし） |
| リスクリワード目標 | 記載なし | 最低1:1.5、理想1:2以上 |

### 設計思想

- エントリーは複数指標のコンフルエンスで判断（維持）
- 損切りは小さく確実に。利確はトレーリングストップで利益を伸ばす
- バックテストで**期待値プラス・PF 1.3以上**を確認してから本番適用する

## 2. スコアリング配点の再設計

### カテゴリ配点

| カテゴリ | 現行 | 新 | 変更理由 |
|---------|------|-----|---------|
| テクニカル指標 | 35点 | **40点** | トレンドフォローの主軸。モメンタム重視 |
| チャート・ローソク足パターン | 25点 | **20点** | パターンは補助。過信は危険 |
| 流動性 | 25点 | **25点** | 維持。約定品質は重要 |
| ファンダメンタルズ | 15点 | **15点** | 維持。品質フィルタとして妥当 |

### テクニカル指標（40点）の内訳

| サブ項目 | 現行 | 新 | 変更内容 |
|---------|------|-----|---------|
| RSI | 13点 | **10点** | 評価基準を逆転: 50-65に最高得点（モメンタム型） |
| 移動平均 | 13点 | **15点** | MA順列（短期>中期>長期）でトレンド強度を評価 |
| 出来高変化 | 9点 | **10点** | 微増。トレンド確認に重要 |
| MACD（新規） | — | **5点** | ヒストグラム上昇・ゴールデンクロスに加点 |

### RSI評価基準の変更

```
Before（逆張り型）:
  RSI 30-40 → 13点（最高）

After（トレンドフォロー型）:
  RSI 50-65 → 10点（最高）  ← モメンタムが乗っている
  RSI 40-50 → 7点           ← トレンド初動の可能性
  RSI 65-75 → 5点           ← 強いが過熱気味
  RSI 30-40 → 3点           ← 下降トレンド中の可能性
  RSI 75+   → 0点           ← 過熱
```

### 維持するもの

- 即死ルール（スプレッド>5%、ボラ異常、株価>¥1,000、決算前後）
- 週足トレンド整合性チェック（-7点）
- 出来高方向性分析（買い集め/投げ売り判定）
- 逆行ウィナーボーナス

## 3. 出口戦略

### 固定利確の廃止 → トレーリングストップ一本化

```
Before:
  利確 = min(レジスタンス, 指値 + ATR×1.5)
  最低保証 = 指値 × 103%（固定3%利確）

After:
  利確 = トレーリングストップのみ（固定TPなし）
```

### トレーリングストップのパラメータ

| パラメータ | 現行（スイング） | 新 | 理由 |
|-----------|----------------|-----|------|
| 初期SL | ATR × 1.0 | **ATR × 1.0**（維持） | 損小は変えない |
| 最大損失 | 3% | **3%**（維持） | リスク管理は堅持 |
| トレーリング発動 | ATR × 0.75上昇後 | **ATR × 0.5上昇後** | より早く発動して利益を守る |
| トレール幅 | ATR × 1.5 | **ATR × 1.2** | やや引き締めて利益確保 |
| 固定TP | 指値 × 103% | **廃止** | 利益の上限を撤廃 |

### タイムストップ

- **最大保有日数: 10営業日**
- 10日経過 → 成行決済（P&L不問）
- 理由: トレンドフォローで10日動かないなら仮説が外れている

### リスクリワード比のエントリーフィルタ（新規）

```
期待RR = (直近レジスタンスまでの距離) / (SLまでの距離)
期待RR < 1.5 → エントリー見送り
```

### 変更しないもの

- 初期SL（ATR×1.0、最大3%）
- ポジションサイジング（リスクベース、資金の2%）
- マーケットレジーム（VIX/日経キルスイッチ）
- ドローダウン管理（週次5%/月次10%/連敗制御）
- セクター集中制限
- ディフェンシブモード

## 4. KPI定義

### 第一KPI（最重要指標）

| 指標 | 定義 | 目標値 |
|------|------|--------|
| **期待値** | (勝率×平均利益) - (敗率×平均損失) | +1.0%以上/トレード |
| **プロフィットファクター(PF)** | 総利益 / 総損失 | 1.3以上 |
| **リスクリワード実績** | 平均利益% / 平均損失% | 1.5以上 |

### 第二KPI（管理指標）

| 指標 | 目標値 | 用途 |
|------|--------|------|
| 勝率 | 40-55%（参考値） | 高すぎたら利確が早すぎる兆候 |
| 最大ドローダウン | 15%以下 | リスク管理の確認 |
| 平均保有日数 | 3-7日 | スイングとして適正か |
| トレード頻度 | 月10-20回 | 機会を拾えているか |

### 廃止する指標

- ~~勝率70%目標~~
- ~~平均利益>3%目標~~

### バックテスト合格基準

```
Before: 勝率70%以上 → 本番適用
After:  PF 1.3以上 AND 期待値 +0.5%以上 AND 最大DD 20%以下 → 本番適用
```

### 日次レポート表示

```
Before: 勝率 XX% | 平均利益 +X.X%
After:  期待値 +X.X% | PF X.XX | RR比 X.XX | 勝率 XX%
```
